## 🚀 راهنمای نصب و افزودن به پروژه (Installation & Setup)

شما به دو روش استاندارد (از روی گیت‌هاب) یا روش توسعه محلی (فضای کاری Go Work) می‌توانید این کتابخانه را به پروژه خود اضافه کنید [1]:

### روش اول: نصب و استفاده استاندارد (Production Mode)

این روش برای زمانی است که کتابخانه روی گیت‌هاب شما منتشر شده است و می‌خواهید آن را به پروژه خود ایمپورت کنید [1]:

۱. ابتدا پوشه پروژه خود را بسازید و مدول گو را در آن مقداردهی اولیه کنید (یوزرنیم گیت‌هاب خود را جایگزین کنید) [1]:
```bash
go mod init mybot
```

۲. با دستور زیر، کتابخانه را از روی گیت‌هاب به پروژه خود اضافه کنید [1]:
```bash
go get github.com/PHX-Go/GoBale
```

۳. اکنون می‌توانید پکیج را در فایل‌های پروژه خود ایمپورت و استفاده کنید [1]:
```go
import (
    "github.com/PHX-Go/GoBale"
    "github.com/PHX-Go/GoBale/models"
)
```

---

### روش دوم: توسعه همزمان محلی با فضای کاری گو (Local Go Workspace - `go.work`)

اگر می‌خواهید کتابخانه اصلی (`gobale`) و ربات خود را به صورت محلی و همزمان توسعه دهید (بدون نیاز به انتشار مداوم روی گیت‌هاب برای تست تغییرات)، استفاده از **Go Workspaces** حرفه‌ای‌ترین روش است [1].

۱. یک پوشه مادر (مثلاً به نام `GoBaleProject`) ایجاد کرده و پوشه‌های `gobale` و ربات خود (مثلاً `mybot`) را در داخل آن قرار دهید [1].

۲. یک فایل به نام **`go.work`** در پوشه مادر بسازید و پوشه‌های مدول‌ها را در آن رجیستر کنید [1]:

```go
go 1.26.3

use (
	./gobale
	./mybot
)
```

۳. همچنین می‌توانید به جای ساخت دستی، فایل فضای کاری را با دستورات زیر در ترمینال پوشه مادر ایجاد کنید [1]:
```bash
go work init
go work use ./gobale
go work use ./mybot
```

#### مزیت عالی روش `go.work`:
با این کار، زمانی که در فایل‌های ربات خود کدهای کتابخانه را با آدرس گیت‌هاب `import "github.com/YOUR_USERNAME/gobale"` صدا می‌زنید، کامپایلر گو به صورت هوشمند متوجه شده و تغییرات را به صورت آفلاین از پوشه محلی `./gobale` می‌خواند [1]. این کار فرآیند تست و توسعه محلی شما را به غایت ساده و سریع می‌کند [1].

# مستندات فنی: دریافت آپدیت‌ها، وب‌هوک و پولینگ (Updates, Webhook & Polling)

این بخش از کتابخانه مسئولیت برقراری ارتباط مداوم، دریافت پیام‌ها و تعاملات خام کاربران از سرور بله و هدایت اتمیک آن‌ها به صف موازی ورکرها را بر عهده دارد [1].

---

## ۱. مدل ساختار داده‌ای آپدیت (`models.Update`)

ساختار `Update` در مسیر `models/types.go` تعریف شده و متناظر با مدل‌های رسمی بله/تلگرام است [1]. هر زمان تعاملی از سمت کاربر با ربات شکل بگیرد، سرور بله اطلاعات آن را در قالب این ساختار ارسال می‌کند [1].

### 📋 فیلدها و مشخصات فیزیکی

| نام فیلد | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `UpdateID` | `int` | شناسه عددی صعودی و یکتا برای رهگیری ترتیب تراکنش‌ها [1]. |
| `Message` | `*Message` | جزییات پیام متنی، رسانه‌ای یا دستوری جدید دریافت شده (اختیاری) [1]. |
| `EditedMessage` | `*Message` | جزییات پیام‌های ویرایش شده (اختیاری) [1]. |
| `CallbackQuery` | `*CallbackQuery` | رویداد کلیک کاربران روی دکمه‌های شیشه‌ای (اختیاری) [1]. |
| `PreCheckoutQuery` | `*PreCheckoutQuery` | درخواست نهایی تایید پیش‌پرداخت‌های مالی (اختیاری) [1]. |

### 💻 نمونه کد استفاده در کانتکست
```go
bot.OnMessage(func(c *gobale.Context) {
    // دسترسی مستقیم به آی‌دی آپدیت جاری در کانتکست
    updateID := c.Update.UpdateID
    fmt.Printf("Update Received: %d\n", updateID)
})
```

## ۲. متد دریافت آپدیت‌ها (`getUpdates`)

متد `getUpdates` وب‌سرویس اصلی پیام‌رسان بله برای دریافت پیام‌ها به روش پادبوم (Polling) است [1]. این درخواست شبکه به صورت Long Polling ارسال شده و کانکشن را تا زمان دریافت پیام جدید در سرور بله باز نگه می‌دارد [1].

### 📋 پارامترهای ورودی (Params)

| نام پارامتر | نوع داده | مقدار پیش‌فرض | توضیحات |
| :--- | :--- | :--- | :--- |
| `offset` | `int` | `-1` | شناسه آپدیت آغازین (برای تایید دریافت پیام‌های قبلی) [1]. |
| `limit` | `int` | `100` | حداکثر تعداد آپدیت‌های دریافتی همزمان در یک پکت (حداکثر ۱۰۰) [1]. |
| `timeout` | `int` | `20` | مدت زمان بازماندن کانکشن در سرور منتظر پیام جدید (ثانیه) [1]. |

### 💻 نمونه کد فراخوانی در لایه کلاینت
```go
params := map[string]any{"offset": offset, "limit": 100, "timeout": 20}
var updates []models.Update

// فراخوانی مستقیم از لایه وب‌سرویس ربات شما
err := b.BaseRequest(ctx, "getUpdates", params, &updates)
```

## ۳. مدیریت وب‌هوک (`setWebhook` ،`deleteWebhook` ،`getWebhookInfo`)

این متدها وظیفه مدیریت وضعیت وب‌هوک (ارسال مستقیم و آنی پیام‌ها توسط بله به وب‌سرور HTTP ربات شما) را بر عهده دارند [1].

### 📋 متدها و شرح عملکرد

* **`setWebhook`**: ثبت و تنظیم آدرس فیزیکی امن سرور شما (HTTPS با گواهی معتبر) روی بله [1].
* **`deleteWebhook`**: حذف و پاک‌سازی هرگونه وب‌هوک فعال روی توکن ربات بله [1]. فراخوانی این متد قبل از شروع متد Polling الزامی است تا تداخل ترافیکی در دریافت پیام‌ها رخ ندهد [1].
* **`getWebhookInfo`**: دریافت اطلاعات و وضعیت وب‌هوک فعال روی توکن بله (شامل آدرس فعال، پیام‌های در صف انتظار شبکه و هشدارهای خطا) [1].

### 💻 نمونه کدهای استفاده در کلاینت
```go
// حذف وب‌هوک فعال قبل از استارت پولینگ
_ = b.BaseRequest(ctx, "deleteWebhook", nil, nil)

// تنظیم وب‌هوک جدید روی سرور امن پروژه شما
params := map[string]any{"url": "https://mydomain.com/webhook"}
_ = b.BaseRequest(ctx, "setWebhook", params, nil)
```

---

## ۴. چرخه پادبوم هوشمند (`RunPolling` / Polling)

متد `RunPolling` موتور محرک و چرخه حیات اصلی ربات در حالت پادبوم است [1]. این متد دارای سه ویژگی امنیتی بسیار پیشرفته در لایه کرنل است [1]:

1. **حذف وب‌هوک پیش از استارت:** جهت ممانعت از تداخل و گم شدن پیام‌ها، در ثانیه اول وب‌هوک فعال را با `deleteWebhook` پاک‌سازی می‌کند [1].
2. **سیستم خاموشی ملایم (Graceful Shutdown):** به محض دریافت سیگنال‌های توقف سیستم‌عامل (`SIGINT` / `SIGTERM`)، به طور موازی کانال صف ورکرها (`close(workerChan)`) را می‌بندد [1]. ورکرها همچنان به پردازش پیام‌های باقیمانده در صف ادامه می‌دهند تا صف کاملاً تخلیه شود [1].
3. **گروه همگام‌سازی ورکرها (`WaitGroup`):** نخ اصلی برنامه با کمک `Wait` مسدود می‌ماند تا آخرین ورکر فعال نیز کار خود را کاملاً تمام کند [1]. در نهایت، تمام سشن‌ها به صورت اتمیک ذخیره شده و ربات به آرامی و بدون از دست رفتن هیچ پیام یا پردازش نصفه‌ای خاموش می‌شود [1].

### 💻 نمونه کد راه‌اندازی
```go
bot := gobale.NewBot("YOUR_BOT_TOKEN", gobale.AutoWorkers)
// ... ثبت روت‌ها ...
bot.RunPolling() // مسدود ماندن نخ اصلی و اجرای چرخه حیات ربات
```

# مستندات فنی: مدل‌های اصلی ربات (Core Models)

این بخش ساختارهای داده‌ای اصلی (Structs) برای مدیریت اطلاعات کاربران، نوع گفتگوها، جزئیات پیام‌های دریافتی و موجودیت‌های متنی (مانند دستورات و لینک‌ها) را توصیف می‌کند [1].

## ۱. مدل کاربر (`models.User`)

این مدل اطلاعات فرستنده پیام یا حساب‌های کاربری متصل به بات (اعم از کاربر حقیقی یا بات‌های دیگر) را کپسوله می‌کند [1].

### 📋 فیلدها و مشخصات فیزیکی

| نام فیلد | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `ID` | `int64` | شناسه عددی یکتا و دائم کاربر در پلتفرم بله [1]. |
| `IsBot` | `bool` | مشخص‌کننده اینکه آیا این کاربر یک ربات است یا خیر [1]. |
| `FirstName` | `string` | نام کوچک کاربر (اجباری) [1]. |
| `LastName` | `string` | نام خانوادگی کاربر (اختیاری) [1]. |
| `Username` | `string` | نام کاربری یا همان آیدی مینی‌مال کاربر بدون کاراکتر `@` (اختیاری) [1]. |
| `LanguageCode` | `string` | کد زبان تنظیم‌شده در کلاینت بله کاربر (مثلاً `fa` یا `en`) [1]. |

### 💻 نمونه کد استفاده در کانتکست
```go
bot.OnMessage(func(c *gobale.Context) {
    if c.Message != nil && c.Message.From != nil {
        user := c.Message.From
        fmt.Printf("فرستنده: %s (@%s)\n", user.FirstName, user.Username)
    }
})
```

## ۲. مدل گفتگو (`models.Chat`)

این مدل اطلاعات پایه‌ای گفتگویی که پیام در آن تبادل شده است (اعم از پی‌وی، گروه، سوپرگروه یا کانال) را نگهداری می‌کند [1].

### 📋 فیلدها و مشخصات فیزیکی

| نام فیلد | نوع داده | توضیحات |
| :--- | :--- | :--- |

| `Title` | `string` | عنوان گروه یا کانال (برای چت‌های شخصی خالی است) [1]. |

```go
bot.OnMessage(func(c *gobale.Context) {

})
```

## ۴. مدل پیام (`models.Message`)

این ساختار، هسته‌ی مرکزی تبادل اطلاعات متنی، رسانه‌ای (عکس، فیلم، سند و...)، موقعیت‌های جغرافیایی، مخاطبین و پرداخت‌های ربات است [1].

```go
bot.OnMessage(func(c *gobale.Context) {

        repliedText := c.Message.ReplyToMessage.Text
        c.Reply(fmt.Sprintf("شما روی پیام %q ریپلای کردید.", repliedText))
    }
})
```

## ۶. موجودیت‌های پیام (`models.MessageEntity`)

این مدل بخش‌های خاصی از یک پیام متنی (مانند دستورات ربات، هشتگ‌ها، لینک‌های وب یا آی‌دی کاربران) را به همراه موقعیت شروع و طول آن‌ها مشخص می‌کند [1].

### 📋 فیلدها و مشخصات فیزیکی

| نام فیلد | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `Type` | `string` | نوع موجودیت؛ شامل مقادیر `"bot_command"` (دستور ربات)، `"mention"` (آیدی کاربر)، `"url"` (لینک وب) [1]. |
| `Offset` | `int` | نقطه شروع فیزیکی موجودیت در داخل متن پیام (شروع از صفر) [1]. |
| `Length` | `int` | طول کاراکترهای موجودیت مورد نظر [1]. |

### 💻 نمونه کد استفاده در کانتکست
```go
bot.OnMessage(func(c *gobale.Context) {
    // اسکن موجودیت‌های پیام برای پیدا کردن دستورات ربات
    for _, entity := range c.Message.Entities {
        if entity.Type == "bot_command" {
            cmd := c.Message.Text[entity.Offset : entity.Offset+entity.Length]
			fmt.Printf("دستور شناسایی شده: %s\n", cmd)
        }
    }
})
```

## ۴. راهنمای جامع روش‌های ۳گانه‌ی ارسال فایل در فریم‌ورک

کتابخانه شما از **هر ۳ روش استاندارد بین‌المللی** برای ارسال تصاویر، اسناد، صداها و سایر فایل‌ها به وب‌سرویس بله پشتیبانی می‌کند [1]. توسعه‌دهنده بر اساس نیاز خود می‌تواند از هر یک از روش‌های زیر استفاده کند [1]:

### روش اول: ارسال از طریق شناسه فایل (`file_id`) - سریع‌ترین و بهینه‌ترین روش
هر فایلی که یک‌بار در سرورهای بله آپلود می‌شود، یک شناسه منحصربه‌فرد به نام `file_id` دریافت می‌کند [1]. شما می‌توانید این آیدی متنی را مستقیماً برای ارسال مجدد به کاربران دیگر بفرستید [1].

* **مزایا:** تحویل آنی (کمتر از صدم ثانیه)، عدم مصرف ترافیک آپلود سرور و بدون نیاز به پردازش دیسک [1].

```go
bot.OnCommand("/resend", func(c *gobale.Context) {
    fileID := "AgADBAADFjg..._file_id_example"
    
    // ارسال فوری بدون مصرف پهنای باند آپلود با پاس دادن مستقیم file_id
    _, _ = c.SendPhoto(fileID, gobale.WithCaption("این تصویر قبلاً آپلود شده است"))
})
```

---

### روش دوم: ارسال از طریق آدرس وب (HTTP URL)
شما می‌توانید آدرس مستقیم وب (لینک مستقیم عکس، فیلم یا داکیومنت موجود در هاست یا سایت خود) را به عنوان ورودی به ربات بدهید [1]. وب‌سرور بله خود به طور خودکار فایل را از هاست شما دانلود کرده و برای کاربر ارسال می‌کند [1].

* **مزایا:** عدم مصرف ترافیک آپلود سرور و بسیار راحت برای محصولات و سایت‌های متصل به دیتابیس [1].

```go
bot.OnCommand("/send_link", func(c *gobale.Context) {
    photoURL := "https://bale.ai/assets/images/logo.png"
    
    // سرور بله خودکار عکس را دانلود کرده و برای کاربر می‌فرستد
    _, _ = c.SendPhoto(photoURL, gobale.WithCaption("تصویر لوگوی مستقیم از سایت ما"))
})
```

---

### روش سوم: ارسال فیزیکی فایل محلی از هارد دیسک سرور (Multipart/Form-Data)
اگر فایل به صورت فیزیکی روی سیستم یا سرور شما قرار دارد (مانند پوشه `assets/pic1.jpg`)، کتابخانه فایل را باز کرده و بایت‌های خام آن را به صورت استریمینگ مالتی‌پارت با متد `POST` به وب‌سرور بله آپلود می‌کند [1].

* **مزایا:** استقلال کامل و امکان آپلود مستقیم هر فایل تولید شده درون‌برنامه‌ای یا محلی بدون نیاز به آپلود قبلی [1].
* **بهینه‌سازی ویژه حافظه رم:** این لایه به جای پر کردن حافظه رم، با استفاده از بافر ایمن از پیش تخصیص داده شده (`bytes.Buffer` حاصل از `sync.Pool`)، حجم اشغال رم را مستقل از حجم فایل به شدت بهینه نگه می‌دارد و از خطای EOF جلوگیری می‌کند [1].

```go
bot.OnCommand("/upload_local", func(c *gobale.Context) {
	localPath := "assets/banner.jpg"
	
	// کتابخانه خودکار مسیر را باز کرده و با متد Multipart به بله آپلود می‌کند
	_, _ = c.SendPhoto(localPath, gobale.WithCaption("آپلود مستقیم پوستر فیزیکی از روی هارد سرور"))
})
```

# مستندات فنی: درگاه پرداخت بانکی و فاکتورها (Bale Payment & Invoice Methods)

## ۴. مدل درخواست پیش‌پرداخت (`models.PreCheckoutQuery`)

وقتی کاربر در کلاینت بله روی دکمه نهایی فاکتور کلیک کرده و رمز دوم خود را وارد می‌کند، بله ابتدا این درخواست پیش‌پرداخت را به ربات می‌فرستد تا ربات موجودی کالا یا صحت فاکتور را برای آخرین بار تایید کند [1].

### 📋 فیلدها و مشخصات فیزیکی

* **`ID`** (`string`): شناسه یکتای تراکنش پیش‌پرداخت (برای ارسال تاییدیه استفاده می‌شود) [1].
* **`From`** (`User`): اطلاعات کاربری که در حال پرداخت است [1].
* **`TotalAmount`** (`int64`): کل مبلغ فاکتور پرداختی به ریال [1].
* **`InvoicePayload`** (`string`): همان آیدی فاکتور اختصاصی که در زمان ارسال فاکتور ست کرده بودید [1].

## ۵. متد تایید پیش‌پرداخت کاربر (`answerPreCheckoutQuery`)

ربات شما موظف است حداکثر ظرف **۱۰ ثانیه** پاسخ تایید (`true`) یا لغو (`false` به همراه متن ارور مانند "اتمام موجودی کالا") را به سرور بله ارسال کند [1].

* **متد کلاینت:** `(b *Bot) AnswerPreCheckoutQuery(preCheckoutQueryID string, ok bool, errMsg string) (bool, error)` [1]
* **متد کانتکست:** `(c *Context) AnswerPreCheckout(ok bool, errMsg string) (bool, error)` [1]

### 💻 مثال استفاده
```go

    log.Printf("درخواست پیش‌پرداخت دریافت شد | فاکتور: %s | مبلغ: %d", query.InvoicePayload, query.TotalAmount)

    // تایید موفقیت‌آمیز بودن و اجازه عبور تراکنش به مرحله نهایی پرداخت بانکی
    _, _ = c.AnswerPreCheckout(true, "")
})
```

## ۶. مدل دریافت تاییدیه پرداخت موفق (`models.SuccessfulPayment`)

پس از تایید پیش‌پرداخت و کسر نهایی پول از حساب بانکی کاربر توسط درگاه بله، بله یک پیام سبز رنگ تاییدیه پرداخت موفق در قالب این ساختار به چت ارسال می‌کند [1]. این پیام حاوی شماره تراکنش رسمی درگاه بله است [1].

### 📋 فیلدها و مشخصات فیزیکی

* **`TotalAmount`** (`int64`): کل مبلغ پرداخت شده به ریال [1].
* **`InvoicePayload`** (`string`): آیدی فاکتور پرداختی سیستم شما [1].
* **`ProviderPaymentChargeID`** (`string`): شناسه رهگیری تراکنش مالی (کد تراکنش رسمی بله/بانک) [1].

### 💻 نمونه کد استفاده در کانتکست
```go

    if c.IsPayment() {
        payment := c.Message.SuccessfulPayment
        log.Printf("💵 پرداخت موفق دریافت شد! کد رهگیری بانک: %s\n", payment.ProviderPaymentChargeID)
```

## ۱. چرخه حیات، همروندی و مدیریت شبکه (Core Concurrency & Networking)

این بخش مسئولیت مدیریت ایمن منابع سرور، ممانعت از کرش‌ها و بهینه‌سازی نرخ درخواست‌ها را بر عهده دارد [1].

* **`StartWorkers(ctx)`**: ساخت و استارت اتمیک ورکرها (نخ‌های همزمان) بر اساس الگوی رِنج کانال گو جهت پردازش موازی و اتمیک آپدیت‌ها [1].
* **`processUpdate(ctx, update)`**: مدیریت استخراج اطلاعات، مپ کردن نشست‌ها، اجرای میدل‌ورها و هندلرهای روت‌ها [1].
* **`SendUpdateToWorkerChan(update)`**: ارسال آپدیت دریافتی به صورت اتمیک به صف ورکرها [1].
* **`RunWebhookTLS(...)`**: راه‌اندازی سرور وب‌هوک با گواهی معتبر فیزیکی TLS و مجهز به تایم‌اوت‌های انقضای کانکشن [1].
* **`generateSelfSignedCert()`**: تولید خودکار گواهی خودامضا شده‌ی TLS در زمان وب‌هوک معمولی ربات [1].
* **`NewCircuitBreaker(...)` (Half-Open CAS Isolation)**: دژکوب هوشمند که فاز ریکاوری نیمه‌باز را ایزوله کرده و با کمک پرچم اتمیک فقط به **یک** درخواست تستی اجازه عبور می‌دهد تا سرور بله در صورت بازیابی، بمباران ترافیکی نشود [1].
* **`NewRateLimiter(...)` (Ticker-Free Token Bucket)**: محدودکننده نرخ درخواست‌ها با تکنیک «پیش‌خرید زمان» بدون ران کردن گوروتین موازی در بک‌گراند (مصرف سی‌پی‌یو ربات در حالت استراحت دقیقاً صفر درصد است) [1].
* **`BaseRequest` و `BaseRequestMultipart` (Retry Backoff)**:
  * دکودر شبکه مجهز به مکانیزم تلاش مجدد نمایی (تا ۳ بار تکرار خودکار در صورت نوسان شبکه یا خطاهای ۵xx بله با فواصل متوالی ۱۰۰ms، ۳۰۰ms و ۹۰۰ms) [1].
  * بازسازی پویای ریدر درخواست (`bytes.NewReader`) در هر تلاش مجدد جهت ممانعت از خطاهای بسته بودن بدنه پکت در گو [1].
* **`backoffDelay(attempt)`**: محاسبه تصاعدی زمان وقفه تلاش مجدد شبکه [1].
* **`NewFileCache(...)` (Persistent GOB Cache)**: ذخیره اتمیک و دائم شناسه‌ی فایل‌های آپلود شده روی هارد دیسک (آپلود مجدد همان فایل محلی را پس از ری‌استارت ربات به صفر می‌رساند) [1].
* **`GetMemoryStats() / SetGCPercent() / SetMemoryLimit()` (Ops Controls)**:
  * `GetMemoryStats`: دریافت آنلاین آمار دقیق رم مصرفی سیستم از داخل بله [1].
  * `SetMemoryLimit`: محدود کردن تهاجمی رم روی رانتایم گو جهت جلوگیری از پر شدن حافظه VPS [1].
  * `SetGCPercent`: تنظیم موازنه مصرف پردازنده و سرعت زباله‌روبی سیستم [1].
* **`GetMetrics()`**: استخراج و نمایش آنلاین نرخ پردازش پیام‌ها، کارهای در صف و راندمان زمانی شبکه [1].

### 💻 نمونه کد مدیریت منابع و مانیتورینگ
```go
bot.OnCommand("/status", func(c *gobale.Context) {
    stats := c.Bot.GetMemoryStats()
    metrics := c.Bot.GetMetrics()
    c.Reply(fmt.Sprintf("💻 رم هیت: %.2f MB\n📊 کل تراکنش‌ها: %d\n⚡ تاخیر شبکه: %v", stats.AllocMegabytes, metrics.TotalUpdates, time.Duration(metrics.NetworkLatencyNs)))
})
```

## ۲. مدیریت نشست‌ها و پایگاه داده محلی (State, Sessions & Local Storage)

## ۴. کیبوردها و صفحه‌بندی هوشمند (Smart Keyboards & Pagination)

* **`Row(buttons ...any)` (دکمه‌های تکی هوشمند)**: پذیرش همزمان رشته متنی ساده، دکمه تکی هوشمند شیشه‌ای (`models.Btn`) یا معمولی (`models.ReplyBtn`) در یک ردیف مشترک [1].
* **`NewPaginatedKeyboard()`**: متد بومی چنک کردن و صفحه‌بندی اسلایس‌های بزرگ کیبورد [1].
* **`SendPaginated(...)`**: صفحه‌بندی استاتیک پیام‌ها با ذخیره‌سازی داده‌ها در کش منقضی‌شونده (TTL) برای ممانعت از لیک حافظه روتر [1].
* **`SendPaginatedDynamic(...)`**: صفحه‌بندی داینامیک دکمه‌ها (فراخوانی دکمه‌ها از دیتابیس به صورت On-Demand فقط برای صفحه جاری) [1].
* **`registerGlobalPagination()`**: متد ثبت خودکارِ یکپارچه سیستم صفحه‌بندی و پاک‌ساز پس‌زمینه با الگوی `sync.Once` بدون نیاز به کانفیگ توسط برنامه‌نویس [1].
* **`NewReplyKeyboardMarkupFromSlice / NewInlineKeyboardMarkupFromSlice`**: تبدیل اسلایس رشته‌ای به ماتریس دو بعدی دکمه‌های معمولی یا شیشه‌ای به صورت دو ستونه یا سه ستونه [1].

### 💻 نمونه کد صفحه‌بندی داینامیک از دیتابیس
```go
bot.OnCommand("/products", func(c *gobale.Context) {
	// واکشی پویای دکمه‌ها فقط به تعداد ۵ عدد در هر صفحه از دیتابیس (Zero-Memory Overhead)
	_, _ = c.SendPaginatedDynamic("🛒 منوی محصولات:", 5, func(page, limit int) ([]models.InlineKeyboardButton, int, error) {
		dbProducts := db.LoadPage(page, limit)
		totalCount := db.TotalCount()

		var names []string
		var callbacks []string
		for _, prod := range dbProducts {
			names = append(names, prod.Name)
			callbacks = append(callbacks, fmt.Sprintf("prod:%d", prod.ID))
		}

		// استفاده از پترن بیلدر بسیار تمیز و تفکیک‌شده‌ی اختصاصی شما
		markup := models.InlineMarkup().
			Row(names...).
			Callbacks(callbacks...).
			Build()

		// استخراج اتمیک اسلایس مسطحِ دکمه‌های ساخته شده از ردیف اول کیبورد
		return markup.InlineKeyboard[0], totalCount, nil
	})
})
```

## ۵. سیستم هوشمند تجزیه آرگومان‌ها (Smart Argument Parsers)

* **`scanValues(args, sep, targets...)`**: هسته اتمیک و سریع گو (تایپ سوئیچ بدون رفلکشن) برای مچ کردن خودکار رشته‌ها به پوینتر متغیرها [1].
* **`ScanArgs(targets...)`**: تجزیه هوشمند آرگومان‌های دستورات متنی (تلفیق با جداکننده اسپیس) با قابلیت تفکیک متن انتهایی خط [1].
* **`ScanCallbackArgs(targets...)`**: تجزیه هوشمند پارامترهای دکمه شیشه‌ای کلیک شده (تلفیق با جداکننده دونقطه) [1].
* **`parseDurationWithDays(s)`**: پارسر هوشمند زمان که علاوه بر واحدهای استاندارد گو، واحدهای **`d` (روز)** و **`w` (هفته)** را به طور خودکار به ساعت تبدیل می‌کند [1].

### 💻 نمونه کد تجزیه آرگومان‌های دکمه شیشه‌ای
```go
bot.OnCallbackData("buy", func(c *gobale.Context) {

    var count int
    // دریافت مقادیر دکمه خرید با فرمت "buy:1029:5" در ۱ خط
    _ = c.ScanCallbackArgs(&productID, &count)
```

## ۸. میان‌افزارها (Middlewares)

* **`Recovery`**: محافظت از هسته ربات در برابر کرش کردن ناشی از پنیک در هندلرها [1].
* **`AdminOnly(adminID)`**: مسدود کردن دستور فقط برای ادمین اصلی ربات [1].

* **`SendAction(action)`**: ارسال خودکار وضعیت در حال تایپ یا آپلود رسانه قبل از شروع متد اصلی روت [1].

* **`MandatoryJoinMulti(alertText, channels...)`**: عضویت اجباری چندگانه هوشمند که فقط لینک کانال‌های عضو نشده را استخراج کرده و به صورت کیبورد شیشه‌ای به کاربر نمایش می‌دهد [1].
* **`ReportErrorToAdmin(botName, targetChatID)`**: سیستم گزارش لحظه‌ای خطاها و پنیک‌های ربات به پی‌وی، گروه یا کانال لاگ اختصاصی به همراه نام اختصاصی ربات به صورت کاملاً لوپ‌سیف و ایمن [1].
* **`ChatRateLimitMiddleware`**: محدود کردن فرکانس پیام‌های ارسالی چت‌ها با الگوریتم سطل توکن [1].
* **`Blacklist / Maintenance / AntiSpam / AntiLink / Logger`**: سایر میدل‌ورهای پایداری و امنیتی توکار ربات [1].
